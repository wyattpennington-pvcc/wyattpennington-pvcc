Version Control System (VCS)

A version control system (VCS) is a tool that helps manage changes to source code over time. It allows multiple developers to collaborate on a project, track changes, and revert to previous versions if needed.

Git

Git is a distributed version control system that allows developers to track changes in their codebase. It was created by Linus Torvalds in 2005.

GitHub

GitHub is a web-based platform that provides hosting for software development using Git. It includes features like code collaboration, bug tracking, and more.

Difference between Git and GitHub

Git is the version control system, while GitHub is a platform that uses Git for version control. Git is used locally on a developer's machine, while GitHub is a remote repository hosting service.

GitHub Founder and History

GitHub was founded by Chris Wanstrath, PJ Hyett, and Tom Preston-Werner in 2008. It started as a way to simplify sharing code and collaborating on software development projects.

Git/GitHub Terms

Repository

A repository (repo) is a location where all the files for a particular project are stored, along with the version history of those files.

Commit

A commit is a snapshot of the changes made to a repository. It represents a specific point in the project's history and includes a message describing the changes.

Fork

Forking a repository means creating a personal copy of someone else's project. It allows you to make changes without affecting the original project, and you can later propose those changes back to the original repository.

Push

Pushing refers to sending your committed changes to a remote repository. It's a way to share your work with others or update the central repository.

Pull Requests

A pull request is a way to propose changes to a repository. It allows others to review and discuss the changes before they are merged into the main project.

Workflows

Workflows in the context of Git/GitHub refer to the defined set of rules or processes that dictate how code changes move from development through testing and eventually to deployment.

Issues

Issues are a way to track tasks, enhancements, bugs, or other kinds of questions within a repository. They can be used for collaboration, discussion, and project management.

Raw Button

The Raw button allows you to view the raw content of a file in its original format, without any formatting applied. It's useful when you need to directly access the file's raw data.

Blame Button

The Blame button, often found in version control systems, shows the revision and author of each line in a file. It helps identify who last modified a particular piece of code.
